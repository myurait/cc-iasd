# rework 00. リワーク検討索引

作成日: 2026-07-03  
状態: 検討中 v0.1

---

## 1. このディレクトリの目的

`docs/development/rework/` は、先行事例調査を踏まえて cc-iasd の設計をリワークするための検討メモを置く。

このディレクトリの文書は検討メモであり、設計正本ではない。採用が確定した項目は `docs/development/` の本編ドキュメント（01〜09、12）と `10_todo.md` へ反映し、rework 側は判断記録として残す。

---

## 2. 文書構成

```text
00_rework_index.md
  リワーク検討の目的、進め方、ステータス

01_prior_art_survey.md
  先行事例調査の正本。spec 駆動開発系、マルチエージェント・ロール系、
  自律実行ガバナンス・証跡系の 3 方向の調査結果、重複度評価、空白地帯

02_adoption_review.md
  「いいとこどり」検討メモ。採用候補ごとに 採用 / 保留 / 不採用 の推奨、
  理由、反映先ドキュメントを記載

03_vocabulary_alignment.md
  語彙統合検討メモ。cc-iasd の語彙と先行事例の語彙を突き合わせ、
  統合（リネーム）/ 維持の推奨と語彙対応表を記載

04_radical_rework.md
  破壊的リワーク提案「cc-iasd kernel」。3 不変条件（src 隔離 / 証跡管理主義 /
  推測補完禁止）を構造で守る決定論的状態機械カーネルとして全体を再設計する提案。
  採用時は本編 01〜13 の大部分と CLI を置き換える

05_work_plan.md
  kernel 移行作業計画。Phase 1（本編正本化）-> Phase 2（P1 実装）->
  Phase 3（README 再構築）の順序、各 Phase の作業項目・完了条件・レビューゲート

06_doc_rework_guide.md
  Phase 1 改稿ガイド。並行改稿の共通規約（用語対応表・章構成割当・
  文書間責務境界・文体規約・grep 検査リスト・逸脱時手順）
```

---

## 3. 進め方

```text
進行手順:
1. 01_prior_art_survey.md に調査結果を正本化する（完了）
2. 02_adoption_review.md に採用候補と推奨判断を記載する（完了）
3. 人間が 02 の推奨判断をレビューし、採用 / 保留 / 不採用 を確定する
4. 採用確定分を本編ドキュメントと 10_todo.md へ反映する
5. 02 の各項目に確定結果と反映先を追記し、記録として閉じる
```
